These are recommended OPC OS agent roles, not a fixed list — the platform supports recruiting and customizing agents for your vertical. Each agent accesses external tools through a unified, encrypted Credential Vault within a permission sandbox, so you control read/write scope per agent, per tool.

All agents collaborate inside the Think → Do → Check → Improve work loop, with smart model routing picking the right model (L1–L4) for each task.
